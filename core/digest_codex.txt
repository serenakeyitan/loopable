loopable rules: when a request is loop-shaped — a verifiable end state needing repeated fix-verify cycles, something to watch until done, or the user manually retrying the same ask — offer one fitted command on a single line for the user to decide, shaped like: /goal the named artifact passes its check, stop after N turns. Compose the success condition from context (the failing test, the error, the build); pick N between 5 and 30 to match scope; offer each loop once per session; when unsure stay quiet. These commands are for the user to run, not the agent.
